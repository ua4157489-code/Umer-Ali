## 🎯 Overview

The SOC Analyst Toolkit is a comprehensive Python-based security monitoring platform that simulates real-world Security Operations Center workflows. It implements core defensive security capabilities including threat intelligence processing, automated log analysis, network vulnerability assessment, and structured incident response.

### Why This Project?

- **Interview Ready**: Demonstrates practical understanding of SOC workflows, SIEM logic, and security automation
- **Production Patterns**: Uses enterprise software architecture (modular design, OOP, type safety)
- **Extensible**: Easy to integrate with real APIs (Splunk, VirusTotal, MISP)
- **Educational**: Clear documentation of security concepts and detection engineering

### Component Breakdown

#### 1. **ThreatIntelligence** (`threat_intel.py`)
**Purpose**: IOC (Indicators of Compromise) management and reputation checking

**Key Features**:
- Malicious IP reputation database
- File hash verification against known malware
- Domain categorization
- Confidence scoring system

**Architecture Pattern**: Repository Pattern - abstracts data source (simulated local DB, extensible to MISP/ThreatStream APIs)

2. LogAnalyzer (log_analyzer.py)
Architecture Pattern: Strategy Pattern - different analysis strategies for different log types (auth, web, system)
State Management: Uses defaultdict(deque) for efficient time-series analysis of failed login attempts without database dependency
Interview Talking Point: "The sliding window approach for brute force detection prevents memory bloat while maintaining detection accuracy. In production, this would feed into a time-series database like InfluxDB."

3. NetworkScanner (network_scanner.py)
Purpose: Vulnerability assessment and asset discovery
Scanning Methodology:
TCP connect scanning (no raw socket privileges required)
Service fingerprinting via port-to-service mapping
Vulnerability correlation with known exploitable services
Risk scoring algorithm: (vulns × 20) + (open_ports × 5)
Security Considerations:
Non-intrusive scanning (no exploitation)
Timeout handling to prevent hanging
Error handling for network segmentation
Interview Talking Point: "This demonstrates understanding of the attack surface management phase in vulnerability management programs, similar to tools like Nessus or OpenVAS."

4. IncidentResponse (incident_response.py)
Purpose: Structured incident management and automated playbook execution

Workflow:
Alert Generated → Incident Created → Playbook Assigned → Response Actions → Resolution

Playbook Examples:
Malware Detection: Isolate → Memory dump → Hash block → IOC sweep
Brute Force: IP block → Log review → Password reset → MFA enforcement
Data Exfiltration: Isolation → Evidence preservation → Scope assessment → Legal notification
Architecture Pattern: State Machine - incidents transition through OPEN → CONTAINED → RESOLVED/ESCALATED
Interview Talking Point: "This implements the NIST Incident Response Lifecycle (Preparation, Detection, Analysis, Containment, Eradication, Recovery) in code, showing understanding of structured security processes."

5. SecurityDashboard (dashboard.py)
Purpose: Central orchestration and metrics aggregation
Responsibilities:
Component initialization and wiring
Metrics collection (alerts, incidents, blocks)
Threat level calculation algorithm
Interactive CLI interface
Continuous monitoring simulation (threading)
Metrics Dashboard:
Real-time threat level assessment (LOW/MEDIUM/HIGH/CRITICAL)
Security posture recommendations
Incident volume tracking
Response effectiveness indicators

Example Workflows
Scenario 1: Malware Investigation
# Analyze suspicious file
toolkit = SecurityDashboard()
result = toolkit.analyze_file("/tmp/suspicious.exe")

# If malicious, automatically creates incident
if result["is_malicious"]:
    print(f"Incident {result['incident_id']} created")
    print("Playbook actions:", result["recommended_actions"])

Scenario 2: Brute Force Response
# Simulate failed login attempts
for _ in range(10):
    alert = toolkit.process_log_entry("auth", 
        "Failed password for admin from 10.0.0.50")

# System auto-detects pattern and creates HIGH severity incident
# Analyst executes containment:
toolkit.incident_response.execute_response("INC-1001", "CONTAIN")

Scenario 3: Vulnerability Assessment
# Scan critical asset
results = toolkit.network_scanner.scan_host("192.168.1.1")

# Review findings
for vuln in results["vulnerabilities_found"]:
    if vuln["severity"] == "HIGH":
        print(f"Critical: {vuln['issue']} on port {vuln['port']}")
